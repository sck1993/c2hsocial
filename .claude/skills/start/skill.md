# /start — 프로젝트 컨텍스트 파악

새 세션에서 작업을 시작하기 전에 프로젝트 전체 구조를 파악한다.
아키텍처·API·데이터모델은 MEMORY.md에 이미 정리되어 있으므로, 파일을 직접 읽는 것은 최소화한다.

## 실행 순서

1. **MEMORY.md 확인** — 세션 시작 시 자동 로드되므로 내용 재확인만 (읽지 않아도 됨)
2. **`functions/src/` 파일 목록** — `ls` 또는 Glob으로 구조 확인 (파일 내용은 읽지 않음)
3. **최근 커밋 10개** — `git log --oneline -10` 으로 최근 작업 파악
4. **`functions/.env` 존재 여부** — 존재 확인만 (내용 절대 읽지 않음)

## 완료 후 출력

파악한 내용을 아래 형식으로 요약:

```
## 프로젝트 현황
- 구현된 플랫폼: ...
- 최근 작업 내역: (git log 기반)
- 신규/변경 파일: (MEMORY.md와 비교해 새로 보이는 파일)
- 특이사항: ...
```

요약 후 "컨텍스트 파악 완료. 작업 지시를 내려주세요." 로 마무리.
